Receipts institutions can defend in audit.

Regulated automation — permitting, claims, compliance — cannot ship on model answers alone. Buyers need a receipt: which fields were extracted, which checks passed, which rules fired, and whether a human must review.

Try Live Runner View Dashboard

The problem

GovOS and InsuranceOS buyers ask the same question: can you prove why the system approved or rejected this case? Black-box LLM outputs fail procurement. Logs are not receipts. Spreadsheet confidence is not auditable.

The product shape

❌ Model answer only

“Approved.” No field-level confidence. No rule trace. No reviewer queue.

✓ Decision receipt

plan → run → doctor → report — staged pipeline, confidence DAG, IF-THEN rules, explicit human-review routing.

Three-layer confidence DAG

  • L1 — Value confidence: per-field extraction scores (blueprint sqft, zone class, claim dates).
  • L2 — Check confidence: regulation checks that depend on L1 nodes (setback, exclusion).
  • L3 — Application confidence: min-propagated score that gates automation vs human review.

Doctor, not hope

brain-receipts doctor flags low confidence, rules version drift, audit trail gaps, and decision mismatches — the same gates a reviewer would check before sign-off. The dashboard surfaces these live in Live Runner.
